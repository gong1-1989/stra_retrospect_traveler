9.8 开发规范与细则

为确保代码质量和开发效率，项目制定了详细的开发规范。开发者必须严格遵守以下规范，确保代码风格统一、结构清晰、易于维护。

常量与魔法数字规范
  • 禁止魔法数字：代码中禁止直接出现未命名的数字常量，所有数值必须定义为具名常量或枚举
  • 常量定义：使用 const / constexpr 定义常量，放在对应的头文件或专门的常量头文件中
  • 枚举替代：状态码、类型标识等使用 enum class 定义，禁止使用整数常量
  • 配置化：可调参数通过配置文件或 QSettings 管理，不硬编码在代码中
  • 例外情况：0、1、-1 等基础计数数字，以及明确数学意义的常量（如 3.14159）可直接使用
  • 命名规范：常量命名使用全大写加下划线，如 MAX_HEALTH、TIME_LIMIT
代码格式规范
  • 缩进：使用4个空格缩进，禁止使用Tab字符
  • 大括号：左大括号不换行，右大括号单独一行；if/for/while必须使用大括号，即使只有一行
  • 行宽：每行不超过120个字符，超过时合理换行
  • 空行：函数之间空一行，逻辑块之间空一行，头文件包含后空一行
  • 头文件包含：按顺序排列：Qt头文件 → 系统头文件 → 项目头文件，每组之间空一行
  • 指针与引用：*和&紧贴类型名，如 QString* name, const int& value
注释规范
  • 文件注释：每个源文件开头必须有文件注释，包含文件名、功能描述、作者、创建日期
  • 类注释：每个类必须有Doxygen格式注释，说明类的功能、用途、注意事项
  • 函数注释：公有函数必须有Doxygen注释，说明功能、参数、返回值、异常
  • 复杂逻辑：复杂算法和关键逻辑必须有行内注释，说明设计思路
  • TODO标记：未完成的代码使用 // TODO: 描述 标记，便于后续追踪

模块化结构优势：

  • 职责清晰：按功能大类拆分目录，每个模块职责单一，便于理解和维护
  • 并行开发：不同模块可由不同开发者并行开发，减少代码冲突
  • 易于测试：每个模块可独立进行单元测试，提高代码质量
  • 可复用性：模块间通过接口交互，便于代码复用和重构
  • Qt Creator支持：CMake自动识别子目录结构，项目树清晰展示

9.8.3 编译构建流程

CMake构建配置
  • 构建系统：使用CMake 3.20+作为构建系统
  • Qt版本：使用Qt 6.10.1，通过find_package(Qt6 REQUIRED)查找
  • C++标准：使用C++17标准，启用所有警告
  • 编译选项：Debug模式启用调试信息和AddressSanitizer，Release模式启用O3优化；MinGW-w64下自动启用静态链接运行库、大目标文件支持等专用优化
  • 模块化：每个模块作为独立的静态库，主程序链接所有模块

mkdir build && cd build

cmake .. -DCMAKE_BUILD_TYPE=Release -DCMAKE_PREFIX_PATH=/path/to/Qt6

&

# 编译项目

cmake --build . -j$(nproc)

&

Qt Creator开发环境
  • IDE推荐：推荐使用Qt Creator作为主要开发IDE
  • 调试器：使用GDB或LLDB进行调试
  • 代码分析：启用Clang-Tidy和Clazy静态代码分析
  • 格式化工具：使用ClangFormat自动格式化代码，确保风格统一

  • 版本控制集成：内置Git支持，可直接在IDE中查看diff、提交代码、拉取推送、查看提交历史
  • 编译输出：编译错误和警告直接显示在编辑器中，点击可跳转到对应代码行

🔧 Qt调试工具
  • Q_ASSERT：在关键位置使用断言，开发模式下捕获错误
  • qDebug()：使用Qt的调试输出，支持Qt类型的格式化输出
  • 日志系统：使用自定义日志系统，分级别输出（Debug/Info/Warning/Error）
  • 对象树：利用Qt对象树机制，自动管理内存，避免内存泄漏

  • 变量查看：实时查看局部变量、成员变量，支持表达式求值、内存查看和对象树浏览

  • 线程调试：支持多线程调试，可切换查看不同线程的状态、堆栈和变量
  • 性能分析器：集成QProfiler，分析CPU占用、内存分配、图形渲染性能瓶颈
  • 内存检测：集成AddressSanitizer，检测内存泄漏、越界访问、使用已释放内存

  • 调试控制台：游戏内置调试控制台，可执行调试命令
  • 显示碰撞盒：调试模式下显示碰撞盒和触发区域
  • 性能统计：实时显示FPS、内存占用、渲染耗时等性能数据
  • 无敌模式：调试模式下可开启无敌模式，方便测试
  • 传送功能：可直接传送到任意关卡或位置

9.8.5 版本控制规范

Git工作流
  • 主分支：main分支始终保持可运行状态，所有稳定代码都直接提交到main分支
  • 功能开发：小型功能直接在main分支开发，大型功能可临时创建feature分支
  • Bug修复：直接在main分支修复，修复后立即提交并测试
  • 版本标签：重要版本发布时打tag标签，便于版本回溯和回滚
  • 远程备份：定期推送到远程仓库（如GitHub/Gitee），防止数据丢失
提交规范
  • 提交频率：小步提交，每个逻辑单元一次提交，避免超大提交
  • 提交信息：使用约定式提交格式：type(scope): description
  • 类型说明：feat（新功能）、fix（修复）、docs（文档）、style（格式）、refactor（重构）、test（测试）、chore（构建/工具）
  • 描述要求：简洁明了，说明做了什么和为什么这么做
  • 提交前检查：提交前必须编译通过、单元测试通过、代码格式化

9.8.6 代码自查规范

  • 提交前自查：每次提交代码前必须完成自查，确保代码质量
  • 自查方法：对照代码规范逐项检查，使用静态分析工具辅助检查
  • 自查重点：代码正确性、性能影响、内存泄漏、可读性、可维护性、是否符合编码规范
  • 自查记录：重要修改记录在开发日志中，便于后续回溯和维护
  • 自查频率：每个功能模块完成后必须自查，每周进行一次全面代码回顾

9.8.7 插件开发规范

为保证插件质量和兼容性，所有插件开发必须遵循以下规范：

插件开发流程
  • 需求定义：明确插件功能、类型、依赖，编写插件设计文档
  • 环境搭建：配置插件开发环境，引入插件SDK和头文件
  • 接口实现：继承对应插件基类，实现所有纯虚接口
  • 功能开发：实现插件核心功能，通过服务接口与核心交互
  • 元数据编写：编写plugin.json，填写完整的插件信息
  • 测试验证：功能测试、兼容性测试、性能测试、安全测试
  • 打包发布：按规范打包插件，提交审核或发布
插件目录结构

每个插件必须遵循统一的目录结构：

  • plugin.json：插件元数据文件，必填
  • bin/：插件动态库文件
  • assets/：插件资源文件（图片、音频、配置等）
  • config/：插件配置文件
  • data/：插件数据文件
  • docs/：插件文档（使用说明、开发文档等）
  • scripts/：插件脚本文件（如支持脚本）
插件接口规范

接口原则：插件只能通过公开接口与核心交互，禁止直接访问核心内部实现，禁止依赖核心的私有头文件。

  • 基类继承：必须继承对应类型的插件基类，实现所有纯虚函数
  • 宏声明：使用Qt Plugin宏声明插件，确保正确导出
  • 服务获取：通过ServiceLocator获取核心服务，禁止直接引用核心模块
  • 事件通信：通过EventManager发送和接收事件，降低耦合
  • 错误处理：所有接口必须处理异常，禁止异常泄漏到核心
  • 日志输出：使用核心提供的日志接口输出日志，便于调试
插件测试要求
  • 功能测试：所有功能必须经过完整测试，确保符合设计需求
  • 兼容性测试：在支持的游戏版本上测试，确保兼容
  • 性能测试：插件不得明显影响游戏帧率和内存占用
  • 安全测试：检查是否存在安全漏洞，如越权访问、内存泄漏
  • 异常测试：测试异常情况，确保插件崩溃不影响主程序
  • 卸载测试：确保插件可正常卸载，卸载后游戏恢复正常
插件发布规范
  • 版本号规范：遵循语义化版本（主版本.次版本.修订号）
  • 打包格式：统一使用zip格式打包，目录结构符合规范
  • 文件命名：插件包命名格式：{plugin_id}_{version}.zip
  • 文档完整：必须包含使用说明、更新日志、已知问题
  • 依赖声明：完整声明所有依赖，包括依赖的版本范围
  • 签名验证：官方插件必须使用官方私钥签名，防止篡改
插件审核标准

提交到官方平台的插件必须通过以下审核：

  • 功能完整：插件功能完整可用，无明显bug
  • 性能达标：不明显影响游戏性能，内存占用合理
  • 安全合规：无恶意代码，不收集用户隐私，不违反安全规范
  • 兼容稳定：兼容声明的游戏版本，运行稳定不崩溃
  • 文档齐全：有完整的使用说明和开发文档
  • 内容合规：不包含违法违规、侵权、低俗内容

9.8.8 Qt Creator 开发指南

充分利用 Qt Creator 集成开发环境的强大功能，大幅提升开发效率和代码质量。Qt Creator 是 Qt 官方 IDE，为 Qt 开发提供了深度优化的工具链和工作流。

🎨 Qt Designer 可视化 UI 设计

Qt Creator 内置 Qt Designer，支持拖拽式可视化 UI 设计，无需手写代码即可创建复杂界面。

  • 拖拽式设计：从组件面板拖拽 QPushButton、QLabel、QTableWidget 等组件到窗体，实时预览效果
  • 属性编辑器：可视化编辑组件属性（文字、颜色、字体、大小、样式表等），即时生效
  • 布局管理器：使用 QVBoxLayout、QHBoxLayout、QGridLayout 实现响应式布局，自动适配窗口大小
  • 信号槽编辑器：可视化连接信号与槽，如按钮点击触发函数，无需手写 connect 代码
  • 伙伴编辑：设置标签与输入框的伙伴关系，支持快捷键聚焦
  • Tab 顺序编辑：可视化调整组件的 Tab 聚焦顺序
  • 样式表编辑：实时预览 QSS 样式效果，支持颜色选择器和字体选择器

开发效率提升：使用 Qt Designer 设计 UI 比纯代码编写快 3-5 倍，且界面效果更直观，便于UI设计与程序开发分离。UI 文件（.ui）会在构建时自动生成 C++ 代码。
