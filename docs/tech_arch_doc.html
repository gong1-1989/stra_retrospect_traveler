9.1.1 Qt Creator 核心优势

Qt Creator 作为 Qt 官方集成开发环境，为游戏开发提供了全方位的工具支持。充分利用其内置功能可大幅提升开发效率，降低技术门槛。

🎨 可视化UI设计（Qt Designer）
  • 拖拽式界面设计：通过 Qt Designer 可视化编辑器，直接拖拽控件创建UI界面，无需手动编写代码
  • 实时预览效果：设计过程中可实时预览界面效果，所见即所得，减少调试时间
  • 信号槽可视化连接：图形化方式连接信号与槽函数，直观清晰，降低出错率
  • 布局管理器：支持水平、垂直、网格、表单等多种布局，自动适配不同窗口大小
  • 样式表编辑：内置QSS样式表编辑器，支持实时预览样式效果
  • UI文件自动编译：.ui文件自动编译为C++代码，无需手动处理
📦 DLL自动管理与部署
  • windeployqt 一键部署：内置 windeployqt 工具，自动检测并复制所有依赖的Qt DLL文件
  • 依赖自动检测：自动分析可执行文件依赖的Qt模块，智能复制所需DLL
  • 插件自动部署：自动部署图片格式插件、平台插件、样式插件等
  • 多配置支持：支持Debug/Release不同配置的部署，自动区分调试版和发布版DLL
  • 翻译文件部署：自动复制Qt翻译文件，支持多语言部署
  • 一键打包发布：配合Qt Installer Framework可生成完整安装包
🔧 集成调试环境
  • GDB/LLDB深度集成：与调试器深度集成，支持断点、单步、变量查看等完整调试功能
  • 条件断点：支持设置条件断点，满足条件时才中断程序执行
  • 数据断点：支持内存数据断点，变量值改变时自动中断
  • 调用堆栈查看：完整的函数调用链，点击可直接跳转到对应源码
  • 内存查看器：直接查看内存数据，支持多种格式显示
  • 线程调试：多线程调试支持，可切换查看不同线程状态
📊 性能分析工具
  • QProfiler集成：内置性能分析器，分析CPU占用、内存分配、渲染性能
  • CPU性能分析：统计函数调用耗时，定位性能瓶颈
  • 内存分析：追踪内存分配和释放，检测内存泄漏

  • AddressSanitizer集成：内置内存错误检测，发现越界访问、使用已释放内存等问题
📝 版本控制集成
  • Git内置支持：完整的Git功能集成，无需切换到外部工具
  • Diff查看：在编辑器中直接查看代码差异，高亮显示修改内容
  • 代码提交：直接在IDE中提交代码，编写提交信息

9.2 核心技术

  • Qt Graphics View（2D渲染）：高性能2D图形渲染，支持数千个图形项的高效管理和碰撞检测
  • 信号与槽机制：Qt核心特性，实现对象间的高效通信和解耦设计
  • 元对象系统：Qt的核心扩展，提供运行时类型信息、动态属性系统和反射能力
  • Qt Quick / QML：用于快速构建游戏UI界面，支持流畅的动画和过渡效果

9.2.1 Qt Graphics View框架

Qt Graphics View是游戏2D/2.5D渲染的核心技术，提供了强大的图形渲染和场景管理能力。

架构特点：基于图形视图架构，分为场景（Scene）、视图（View）、图形项（Item）三层性能优势：使用BSP树索引，支持数千个图形项的高效管理和碰撞检测坐标系统：支持2D坐标变换，可轻松实现缩放、旋转、平移等效果渲染优化：支持视口裁剪、局部刷新等优化技术，提升渲染效率扩展能力：支持自定义图形项，可实现复杂的游戏对象和特效

9.2.2 信号与槽机制

信号与槽是Qt的核心特性，实现了对象间的高效通信和解耦设计。

工作原理：当对象状态改变时发射信号，槽函数响应信号并执行相应操作类型安全：信号与槽的参数类型必须匹配，编译期检查确保类型安全灵活连接：一个信号可以连接多个槽，多个信号也可以连接同一个槽解耦设计：发送者和接收者互不感知，降低模块间的耦合度线程支持：支持跨线程的信号槽通信，简化多线程编程

9.2.3 元对象系统

Qt的元对象系统提供了运行时类型信息、动态属性系统和反射能力。

核心功能：提供运行时类型信息（RTTI），支持动态属性和方法调用MOC编译器：通过元对象编译器（MOC）预处理代码，生成元对象代码动态属性：支持在运行时为对象添加和查询属性反射能力：可以通过名称调用对象的方法，查询对象的属性和信号脚本支持：为QML和脚本交互提供了基础支持

9.2.4 Qt Quick与QML

Qt Quick / QML用于快速构建游戏UI界面，支持流畅的动画和过渡效果。

声明式语法：使用类似JSON的QML语法，简洁直观地描述UI界面动画框架：内置丰富的动画类型，支持属性动画、过渡动画、粒子效果组件化：支持自定义组件，可复用UI元素，提高开发效率性能优化：使用场景图渲染，GPU加速，保证UI流畅运行交互能力：支持触摸、鼠标、键盘等多种输入方式

9.2.5 Qt Multimedia

Qt Multimedia提供了完整的音频和视频处理能力，是游戏音频系统的基础。

音频播放：支持多种音频格式的播放，包括背景音乐和音效音频输出：支持音量控制、声道控制、音效处理等视频播放：支持视频播放和截图，可用于游戏过场动画相机支持：支持摄像头访问，可用于AR等功能跨平台：在各平台上使用原生音频后端，保证兼容性和性能

9.3 性能目标

  • PC：1080p/60fps（推荐配置），720p/30fps（最低配置）
  • 分辨率：支持4K分辨率，但2.5D视角下1080p为最佳体验
  • 加载时间：初始加载 < 10秒，关卡切换 < 2秒
  • 内存占用：最低2GB，推荐4GB

9.3.1 帧率与分辨率目标

游戏的帧率和分辨率目标根据不同配置分级设定，确保不同硬件水平的玩家都能获得良好体验。

最低配置：720p分辨率，稳定30fps，适合低端PC和笔记本推荐配置：1080p分辨率，稳定60fps，主流游戏PC水平高配支持：支持2K/4K分辨率，高配置PC可获得更清晰的画面帧率自适应：支持帧率动态调整，在复杂场景下保持流畅垂直同步：支持开启/关闭垂直同步，减少画面撕裂

9.3.2 加载时间目标

快速的加载时间是良好游戏体验的重要保障，游戏在加载方面设定了严格的目标。

初始加载：游戏启动到主菜单 < 10秒关卡切换：不同场景之间切换 < 2秒存档加载：读取存档文件 < 1秒资源预加载：关键资源预加载，减少游戏中的卡顿异步加载：支持异步资源加载，不阻塞游戏主线程

9.3.3 内存占用目标

合理的内存占用确保游戏能在各种配置的PC上流畅运行。

最低配置：运行时内存占用 < 2GB推荐配置：运行时内存占用 < 4GB显存占用：最低512MB，推荐1GB以上内存优化：资源自动回收，长时间游戏内存不持续增长64位支持：原生64位程序，支持大内存寻址

9.3.4 CPU与GPU占用目标

合理的CPU和GPU占用确保游戏不会过度消耗系统资源。

CPU占用：推荐配置下CPU平均占用 < 30%GPU占用：推荐配置下GPU平均占用 < 60%多线程优化：充分利用多核CPU，主要逻辑分布到多个线程渲染优化：减少不必要的绘制调用，提升渲染效率后台运行：游戏在后台时自动降低帧率和资源消耗

9.3.5 存储占用目标

游戏的存储空间占用控制在合理范围内，方便玩家下载和安装。

安装包大小：PC版安装包 < 5GB安装后大小：安装完成后 < 8GBDLC预留：预留足够空间用于后续DLC内容增量更新：支持增量更新，减少更新下载量压缩技术：使用高效的资源压缩技术，减小文件体积

9.4 加密JSON存档实现方案

使用Qt原生库实现加密JSON存档系统，确保存档安全和完整性。

9.4.1 数据格式

  • JSON格式：使用QJsonDocument和QJsonObject序列化游戏数据
  • 数据结构：分层JSON结构，按系统分类存储数据
  • 版本兼容：包含版本号，支持旧版本存档的迁移和升级

9.4.2 加密算法

  • AES-256加密：使用QCryptographicHash和QAESEncryption实现AES-256加密
  • 密钥派生：使用PBKDF2算法从密码派生加密密钥
  • 盐值：每个存档文件使用随机盐值，增加破解难度

9.4.3 密钥管理

  • 主密钥：存储在可执行文件中，使用混淆技术保护
  • 设备指纹：结合设备硬件信息生成设备密钥，防止存档复制
  • 用户密码：可选用户自定义密码，进一步增强安全性

9.4.4 完整性校验

  • SHA-256哈希：使用QCryptographicHash计算存档数据的SHA-256哈希
  • 校验流程：读取存档时先验证哈希，确保数据未被篡改
  • 损坏处理：校验失败时自动从备份恢复，或提示存档损坏

9.4.5 多槽位存档

  • 槽位管理：使用QSettings管理存档槽位元数据
  • 自动备份：每次保存前自动备份上一个版本
  • 存档缩略图：保存游戏截图作为存档预览图

安全注意：所有加密和哈希操作均使用Qt原生库，不依赖第三方加密库。密钥存储采用多层保护机制，防止逆向工程。

9.5 技术架构设计

9.5.0 架构模式选型

项目采用模块化+插件化混合架构，结合两者优势，既保证核心系统的性能和稳定性，又提供内容层的灵活性和可扩展性。

🏗️ 模块化（核心层）

实现方式：编译为动态链接库（.dll/.so/.dylib），启动时加载

  • 高性能：编译时优化，无运行时开销
  • 调试方便：可直接调试，符号完整
  • 依赖清晰：明确的模块依赖关系
  • 类型安全：编译期类型检查

适用：渲染引擎、音频系统、输入系统、数据管理、核心游戏逻辑

🔌 插件化（内容层）

实现方式：运行时动态加载插件（Qt Plugin机制）

  • 高度灵活：无需重新编译主程序即可增减功能
  • 支持MOD：玩家和第三方开发者可创建插件扩展内容
  • 热插拔：开发模式支持运行时重载插件
  • 隔离性好：插件崩溃不影响核心程序

适用：关卡内容、敌人配置、装备道具、扩展玩法、MOD

9.5.2 核心类设计

  • GameEngine：游戏引擎主类，管理主循环和各模块
  • SceneManager：场景管理器，负责场景切换和加载
  • Player：玩家类，管理玩家状态和能力
  • TimeSystem：时间褶皱系统，管理时间线和时间操作
  • ResonanceSystem：星核共鸣系统，管理共鸣和环境操控
  • EcosystemSystem：生态共生系统，管理生物和共生关系
  • SaveSystem：存档系统，负责数据的保存和加载
  • ResourceManager：资源管理器，负责资源的加载和缓存
  • PluginManager：插件管理器，负责插件发现、加载、卸载、生命周期管理和热重载
  • IPlugin：插件基类接口，定义所有插件必须实现的统一接口规范
  • ServiceLocator：服务定位器，为插件提供访问核心系统服务的统一入口
  • EventManager：事件管理器，实现插件间、插件与核心间的事件驱动通信
  • ExtensionRegistry：扩展点注册表，管理核心扩展点和插件扩展实现

9.5.3 设计模式

  • 单例模式：管理器类使用单例模式，确保全局唯一访问
  • 观察者模式：事件系统使用观察者模式，实现模块间解耦
  • 状态模式：游戏状态、玩家状态使用状态模式管理
  • 工厂模式：游戏对象创建使用工厂模式
  • 策略模式：AI行为、战斗算法使用策略模式

9.5.4 插件系统设计

基于Qt Plugin机制构建完整的插件系统，支持运行时动态加载、热重载、依赖管理和沙箱隔离，为游戏内容扩展和MOD支持提供基础设施。

插件接口设计

所有插件必须实现统一的IPlugin接口，确保插件与核心程序的解耦：

  • 基础接口：initialize()、shutdown()、getName()、getVersion()、getAuthor()、getDescription()
  • 生命周期：onLoad()加载时调用、onUnload()卸载时调用、onEnable()启用、onDisable()禁用
  • 依赖声明：getDependencies()声明依赖的其他插件，支持版本号约束
  • 扩展点：registerExtensions()向核心注册扩展点，供其他插件调用
插件管理器

PluginManager作为插件系统的核心，负责所有插件的生命周期管理：

  • 插件发现：扫描指定目录，自动发现可用插件
  • 元数据解析：读取plugin.json元数据，验证插件信息
  • 依赖解析：拓扑排序，按依赖顺序加载插件
  • 生命周期管理：加载、初始化、启用、禁用、卸载
  • 热重载支持：开发模式下支持运行时重新加载插件
  • 沙箱隔离：插件运行在独立上下文，异常不影响核心
  • 权限控制：细粒度权限控制，限制插件访问范围
  • 插件查询：提供API查询已加载插件、获取插件实例

插件元数据

每个插件目录下必须包含plugin.json元数据文件，描述插件基本信息：

  • 基本信息：id、name、version、author、description、website
  • 类型声明：type（content/gameplay/ui/render/tool/mod）
  • 依赖声明：dependencies数组，每个依赖包含id和version范围
  • 兼容版本：game_version，声明兼容的游戏版本
  • 权限声明：permissions数组，声明需要的权限
  • 入口点：main，插件动态库文件名
插件安全与沙箱

安全第一：插件系统必须有完善的安全机制，防止恶意插件破坏游戏或系统。

  • 权限系统：细粒度权限控制，插件只能访问声明过的权限范围
  • 异常隔离：插件异常被捕获，不导致主程序崩溃
  • 内存限制：可配置插件最大内存使用，防止内存泄漏影响主程序
  • 文件系统沙箱：插件只能访问指定目录，不能随意读写文件
  • 网络隔离：默认禁止插件访问网络，需显式声明权限
  • 签名验证：官方插件支持数字签名验证，防止篡改
插件热重载

开发模式下支持插件热重载，大幅提升开发效率：

  • 自动检测：监控插件目录，检测到文件变化自动触发重载
  • 状态保存：重载前保存插件状态，重载后恢复状态
  • 平滑过渡：重载过程中游戏不中断，玩家无感知
  • 错误回滚：新版本插件加载失败时，自动回滚到旧版本
  • 调试支持：提供调试接口，可手动触发重载、查看插件状态
插件与核心交互

插件通过核心提供的服务接口与游戏交互，不直接依赖具体实现：

  • 服务定位器：核心提供ServiceLocator，插件通过它获取各种服务
  • 事件系统：插件可监听游戏事件，也可发送自定义事件
  • 扩展点机制：核心定义扩展点，插件可注册扩展实现
  • 数据访问：插件可通过统一API访问游戏数据，如玩家状态、世界数据
  • UI集成：插件可向游戏UI添加自定义界面元素

9.5.5 架构模式实施要点

项目采用模块化为主、插件化为辅的混合架构模式，兼顾性能稳定性与灵活扩展性。

模块化与插件化对比

耦合度

加载方式

性能

MOD支持

需要额外设计MOD接口

天然支持，插件即MOD

为什么选择混合架构

核心理念：稳定的基础用模块化保证性能，灵活的玩法用插件化保证扩展性

  • 核心系统采用模块化：渲染引擎、音频系统、输入系统、数据管理、UI框架等基础系统作为静态模块，确保性能最优和稳定性最高，减少运行时不确定因素
  • 玩法系统采用插件化设计：时间褶皱系统、星核共鸣系统、生态共生系统等核心玩法设计为可插拔模块，便于独立开发、测试和迭代调整
  • 关卡与内容支持插件加载：星球关卡、敌人配置、装备道具等内容可通过插件形式动态加载，便于DLC发布和MOD支持
  • 充分利用Qt原生能力：使用Qt的模块化系统（Qt Modules）组织基础功能，使用Qt Plugin框架实现插件化扩展，无需额外造轮子
模块化实施要点
  • 模块边界清晰：每个模块只负责一个明确的功能领域，模块间通过接口通信，不直接依赖实现
  • 依赖单向：上层模块可依赖下层模块，下层模块不依赖上层模块，避免循环依赖
  • 接口稳定：模块接口一旦确定尽量保持稳定，变更需经过评审，确保协作效率
  • 独立测试：每个模块都有独立的单元测试，可单独验证功能正确性
插件化实施要点
  • 接口先行：先定义清晰的插件接口（纯虚函数），再开发具体插件实现
  • 版本兼容：插件接口设计考虑版本兼容性，新增功能通过扩展接口实现，不破坏原有接口
  • 沙箱机制：第三方MOD插件运行在受限环境中，防止恶意代码影响游戏稳定性
  • 插件管理器：统一的插件管理器负责加载、卸载、生命周期管理和依赖解析

9.6 配置文件设计

游戏采用数据驱动的设计理念，所有游戏数据均通过JSON配置文件管理，便于策划调整和MOD扩展。配置文件涵盖游戏设置、角色数据、敌人数据、物品装备、关卡数据、星核数据等六大类。

详细配置模板请参考9.11配置文件模板章节，包含完整的CMake、JSON、qrc等配置文件模板。

  • game.json - 游戏默认配置（图形、音频、控制、游戏性、UI等）
  • characters.json - 角色数据（玩家属性、成长率、同伴信息）
  • enemies.json - 敌人数据（普通敌人、Boss、AI行为）
  • items.json - 物品装备数据（武器、防具、饰品、消耗品、材料）
  • levels.json - 关卡数据（星球、区域、任务、收集品）
  • starcore.json - 星核数据（星核类型、共鸣机制、能力解锁）

详细说明： 配置文件的详细字段说明、类型、默认值详见9.11配置文件模板章节

游戏全部使用 Qt 原生功能自动管理配置，无需手动编写配置文件读写逻辑。默认配置内置在 Qt 资源系统中，用户配置由 QSettings 自动存储在系统配置目录，完全由 Qt 框架负责管理。

9.6.3 QSettings 用户配置

QSettings - 用户配置自动管理

用户设置全部使用 QSettings 管理，Qt 自动处理存储位置和格式：

  • 自动存储位置：Windows 存在注册表，Linux/macOS 存在系统配置目录，无需手动指定路径
  • 自动格式处理：Qt 自动处理配置的读写和序列化，支持字符串、数字、布尔值等
  • 自动分组管理：使用 beginGroup/endGroup 对配置分组，结构清晰
  • 自动版本兼容：Qt 自动处理配置的版本迁移和兼容性

使用示例：

QSettings settings("StarRetrospect", "Game");

settings.setValue("graphics/fullscreen", true);

bool fullscreen = settings.value("graphics/fullscreen", false).toBool();

完全由 Qt 自动管理，无需编写任何文件读写代码。

9.6.4 存档文件管理

配置文件使用说明：

  • 编辑工具：所有 JSON 配置文件可直接用 Qt Creator 打开编辑，支持语法高亮和格式化
  • 热重载：开发模式下可实现配置热重载，修改后无需重新编译
  • 验证工具：建议编写配置验证脚本，确保 JSON 格式和数据结构正确
  • 版本控制：配置文件纳入 Git 版本控制，便于追踪变更和回滚
  • 扩展方式：新增内容只需在对应 JSON 文件中添加条目，无需修改代码
  • 自动路径定位：使用 QStandardPaths::writableLocation 自动获取系统标准存档目录
  • 自动目录创建：Qt 自动创建存档目录，无需手动检查和创建
  • 加密存储：存档内容使用简单加密，由 Qt 的 QCryptographicHash 辅助处理
  • 自动备份：保存前自动备份上一版存档，防止损坏
  • 跨平台兼容：Qt 自动处理不同平台的路径和格式差异

9.6.5 配置加载机制

配置加载完全由 Qt 自动处理：

  • 默认配置：从 Qt 资源系统读取，程序启动时自动加载
  • 用户配置：QSettings 自动读取，首次运行使用默认值
  • 配置覆盖：用户配置自动覆盖默认配置，由 QSettings 的 value() 函数处理
  • 热加载：开发模式下可重新加载资源，修改配置后立即生效
  • 错误处理：配置损坏时自动使用默认值，Qt 自动处理异常情况

9.7.2 Qt 资源系统结构

resources.qrc - Qt 资源文件

所有资源通过 .qrc 文件组织，编译时自动打包进可执行文件：

<RCC>

&nbsp;&nbsp;<qresource prefix="/images">

&nbsp;&nbsp;&nbsp;&nbsp;<file>images/characters/player.png</file>

&nbsp;&nbsp;&nbsp;&nbsp;<file>images/ui/menu.png</file>

&nbsp;&nbsp;&nbsp;&nbsp;<file>images/icons/star.png</file>

&nbsp;&nbsp;</qresource>

&nbsp;&nbsp;<qresource prefix="/audio">

&nbsp;&nbsp;&nbsp;&nbsp;<file>audio/music/theme.mp3</file>

&nbsp;&nbsp;&nbsp;&nbsp;<file>audio/sfx/click.wav</file>

&nbsp;&nbsp;</qresource>

&nbsp;&nbsp;<qresource prefix="/fonts">

&nbsp;&nbsp;&nbsp;&nbsp;<file>fonts/main_font.ttf</file>

&nbsp;&nbsp;</qresource>

&nbsp;&nbsp;<qresource prefix="/config">

&nbsp;&nbsp;&nbsp;&nbsp;<file>config/game.json</file>

&nbsp;&nbsp;&nbsp;&nbsp;<file>config/levels.json</file>

&nbsp;&nbsp;</qresource>

</RCC>

使用时直接通过资源路径访问，Qt 自动处理加载：

QPixmap pixmap(":/images/characters/player.png");

QSoundEffect sound; sound.setSource(QUrl("qrc:/audio/sfx/click.wav"));

完全无需手动管理文件路径和加载逻辑。

9.7.3 资源自动管理特性

  • 自动编译打包：CMake 配置 AUTORCC 后，Qt 自动编译 .qrc 文件，资源打包进可执行文件
  • 自动加载缓存：Qt 自动管理资源的加载和缓存，首次访问时自动加载，后续直接从缓存读取
  • 自动内存管理：Qt 自动管理资源内存，无需手动释放，资源在程序退出时自动清理
  • 自动格式支持：Qt 自动支持多种图片、音频格式，无需手动编解码
  • 自动路径处理：Qt 自动处理跨平台路径差异，资源路径统一使用 :/ 前缀

9.7.4 资源加载策略

📥 核心资源（内置）
  • UI 资源：全部内置在 .qrc 中，启动时自动加载
  • 核心配置：游戏默认配置全部内置，随程序启动
  • 常用图标：小图标和常用图片全部内置
  • 字体文件：游戏字体全部内置，自动加载
  • 样式表：UI 样式全部内置，启动时自动应用
📤 大型资源（外部）
  • 场景大图：大型场景图片作为外部文件，按需加载
  • 背景音乐：长音频作为外部文件，播放时加载
  • 视频资源：过场动画等大型视频外部存储
  • DLC 资源：DLC 内容作为外部资源包，单独分发
  • Mod 资源：玩家 Mod 资源外部加载，不修改主程序

9.7.5 外部资源管理

  • 自动路径定位：使用 QStandardPaths 自动获取资源目录，跨平台兼容
  • 自动目录扫描：Qt 自动扫描资源目录，发现可用资源
  • 资源前缀系统：使用 qRegisterResourceData 动态加载外部 .rcc 资源包
  • DLC 资源包：DLC 资源编译为独立的 .rcc 文件，运行时动态加载
  • 资源热重载：开发模式下可重新加载资源，修改后立即生效

9.7.6 资源管理器简化设计

由于 Qt 资源系统已经自动处理了大部分资源管理工作，资源管理器只需做简单封装：

  • 便捷访问接口：提供统一的资源访问接口，封装资源路径前缀
  • 错误处理：资源加载失败时提供默认占位资源，避免游戏崩溃
  • 外部资源加载：管理外部资源文件和 DLC 资源包的动态加载
  • 调试功能：开发模式下可查看资源列表和加载状态
  • 多语言支持：配合 Qt 国际化系统，自动切换语言资源

核心的资源加载、缓存、内存管理全部由 Qt 自动处理，无需重复造轮子。

  • LRU缓存：资源使用LRU缓存策略，内存不足时卸载最久未使用的资源
  • 强制回收：切换大场景时强制进行垃圾回收
  • 内存监控：实时监控内存使用，超过阈值自动卸载
  • Qt资源系统：使用Qt的qrc资源系统将资源编译进二进制文件
  • 图片压缩：使用PNG无损压缩，根据平台选择合适的压缩级别

  • 加密保护：重要资源文件进行简单加密，防止资源被轻易提取

资源管理器是游戏的核心模块之一，负责统一管理所有游戏资源。

  • 单例模式：全局唯一的资源管理器实例
  • 类型安全：模板化的资源获取接口，确保类型安全
  • 错误处理：资源加载失败时提供默认占位资源，避免游戏崩溃
  • 调试功能：开发模式下可查看资源加载状态、内存占用、缓存命中率
  • 热重载：开发模式支持资源热重载，修改资源后立即在游戏中看到效果
